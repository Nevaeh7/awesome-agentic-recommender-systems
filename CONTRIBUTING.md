# Contributing

Thanks for improving this list.

## Entry Criteria

A paper or repository should be relevant to at least one of these areas:

- Agentic recommender systems.
- LLM-based recommendation agents.
- Multi-agent recommendation or conversational recommendation.
- User simulation, reward modeling, or benchmark design for recommendation agents.
- Safety, robustness, privacy, fairness, or governance for agentic recommenders.
- Agent recommendation, when the target item is an AI agent rather than a conventional product or content item.

## Format

Use the existing table format:

```markdown
| # | Year | Title | Paper | Code | Venue / Status | Topic | Why relevant |
|---|---:|---|---|---|---|---|---|
| 1 | 2026 | Paper title | [link](PAPER_URL) | [code](CODE_URL) | Venue / status | Topic | One-sentence relevance. |
```

Rules:

- In the `Paper` column, display `[link](...)`, not the raw URL.
- In the `Code` column, display `[code](...)` for public code.
- Use `N/A` if public code is unavailable.
- Prefer official, project-named, or author-linked code repositories.
- Keep the relevance note short and factual.

## Pull Request Checklist

- The paper link resolves.
- The code link resolves, or the code column is `N/A`.
- The venue or status is stated honestly.
- The entry is placed in the most specific section.
- The table remains valid Markdown.
